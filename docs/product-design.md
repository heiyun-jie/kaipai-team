## 一、产品概述

### 1.1 产品定位

「开拍了」= 一个短剧跑龙套演员服务平台（小程序端）。

类比模型：**美团外卖** — 平台统一发布通告、演员投递接活、完成拍摄累积信用。

小程序端只服务演员，平台运营通过管理后台完成。
核心链路：**看通告 + 投递 + 完成拍摄 + 信用积累 + 分享名片**。

### 1.2 V1 目标

跑通演员侧完整闭环链路：

```
注册登录 → 完善演员档案 → 浏览平台发布的通告 → 投递 → 完成拍摄 → 信用积累
                ↕                                              ↓
          美化档案 ← ─ ─ ─ ─ ─ 分享演员明信片 ← ─ ─ ─ ─ 信用证明
```

满足以下条件即可上线内测：
1. 演员能注册登录并完善资料
2. 演员能浏览平台发布的通告并投递
3. 演员能美化档案并分享个人明信片
4. 信用积分体系基础功能可用（预留）

### 1.3 业务模型（V1.1 变更）

> **重要变更**：通告发布权从"剧组自助发布"改为"平台统一发布"。
> 小程序端仅服务演员用户，剧组/运营通过管理后台操作。

| V1 原模型 | V1.1 新模型 |
|-----------|------------|
| 剧组注册 → 自助发项目/角色 | 平台后台统一发布通告/角色 |
| 剧组筛选演员投递 | 平台审核/分配 |
| 小程序含剧组端 | 小程序纯演员端 |

### 1.4 当前不做的事

| 砍掉的功能 | 原因 |
|------------|------|
| 聊天系统 | 线下联系方式足够 |
| 在线支付 | 个人主体小程序不支持 |
| 点赞评论 | 不是社区产品 |
| 复杂推荐算法 | 用户量不够，没数据 |
| 实名认证全套流程 | 增加注册门槛 |
| 消息推送中心 | 后续版本再做 |
| 社交动态社区 | 不是社交产品 |
| 剧组端小程序页面 | 已迁移至平台管理后台（代码保留，不删除） |

### 1.5 目标用户

| 角色 | 描述 | 核心诉求 |
|------|------|----------|
| 跑龙套演员/素人 | 想参与短剧、广告、影视拍摄的个人 | 快速发现机会、一键投递、展示自己、积累信用 |
| 平台运营 | 对接剧组需求并发布通告的平台方 | 通过管理后台发布通告、审核投递、评价演员 |

### 1.6 平台规则

- 小程序端所有用户均为演员身份
- 通告/角色由平台后台统一发布
- 投递通过后显示联系方式，线下沟通
- 不含支付功能
- 演员信用积分通过完成拍摄、获得评价等方式积累

## 二、用户角色与权限

### 2.1 演员端（小程序）

| 功能 | 说明 |
|------|------|
| 注册/登录 | 手机号 + 验证码 |
| 完善个人资料 | 头像、姓名、性别、年龄、身高、城市、自我介绍 |
| 上传素材 | 分类照片（形象照/生活照/剧照）、视频简历 |
| 选择擅长类型 | 古装、现代、反派、喜剧等标签 |
| 完善形象标签 | 体型、发型、语言/方言 |
| 添加作品经历 | 项目名、角色名、拍摄时间、剧照 |
| 浏览通告 | 首页通告流（平台发布），轻量搜索筛选 |
| 查看角色详情 | 角色要求、项目信息 |
| 一键投递 | 提交个人档案 |
| 查看投递记录 | 待审核/通过/拒绝状态 |
| 查看信用积分 | 积分明细、等级、排行榜 |
| 分享名片 | 生成明信片分享到微信 |

### 2.2 平台运营端（管理后台，不在小程序内）

> 剧组端功能已迁移至平台管理后台。小程序内剧组端页面代码保留但不再新增功能。
> 后台设计文档另行编写，不在本文档范围内。

| 功能 | 说明 |
|------|------|
| 发布项目/角色 | 原剧组自助发布，现由平台统一发布 |
| 审核投递 | 查看演员投递、标记通过/拒绝 |
| 评价演员 | 拍摄完成后对演员打分 |
| 查看演员档案 | 查看演员完整资料 |
| 查看演员档案 | 查看投递演员的完整资料 |

### 4.1 登录页 `pages/login/index`（公共页 1/2）

**用途：** 用户通过手机号 + 短信验证码登录，未注册自动注册。

**布局描述：**
```
┌──────────────────────────────┐
│  ████████████████████████████│  ← 深色背景 #121214 全屏
│                              │
│        [Logo 图标]            │
│        「开拍了」              │
│     "演员与剧组的桥梁"         │
│                              │
│  ┌──────────────────────┐    │
│  │ 📱 请输入手机号        │    │  ← 深色区输入框（半透明底）
│  └──────────────────────┘    │
│  ┌────────────────┬─────┐    │
│  │ 请输入验证码     │获取  │    │
│  └────────────────┴─────┘    │
│                              │
│  ┌──────────────────────┐    │
│  │     登录 / 注册       │    │  ← 聚光橙按钮
│  └──────────────────────┘    │
│                              │
│  ☑ 已阅读并同意《用户协议》    │
│                              │
│  ── 或 ──                    │
│  [微信一键登录]               │  ← 玻璃质感按钮
│                              │
└──────────────────────────────┘
```

**数据字段：**
- 手机号（11位）
- 短信验证码（6位）

**交互行为：**
- 点击「获取验证码」→ 60秒倒计时
- 登录成功 → 已有身份直接进首页，无身份跳转身份选择页
- 微信一键登录 → 获取微信手机号快捷登录

**视觉提示：**
- 全屏深色背景 #121214，营造影院沉浸感
- Logo 白色/聚光橙，居中偏上，占页面上方 30%
- 输入框：`rgba(255,255,255,0.1)` 底，无边框，白色文字
- 登录按钮：聚光橙 #FF6B35 实心，高度 96rpx
- 微信登录按钮：玻璃质感 `rgba(255,255,255,0.15)` + backdrop-blur

## 五、全局 UI 规范 — 电影感沉浸式玻璃拟态风格 (Cinematic Glassmorphism)

> 在保持专业工具高效性的基础上，通过光影与材质的运用，营造高端且充满影视行业艺术气息的视觉氛围。

### 5.1 设计风格定义

**风格关键词：** 沉浸、电影感、玻璃拟态、聚光灯、先锋

**设计理念：**
- 模拟影院黑暗环境的沉浸感，用深色背景衬托彩色内容
- 聚光灯橙作为品牌色，象征舞台聚光灯和创作热情
- 玻璃拟态（Glassmorphism）增加界面通透性和层级感
- 整体视觉对标 Netflix、IMDb 等国际影视娱乐平台，而非传统白底蓝条的行政风格
- 充分考虑微信小程序原生胶囊按钮的适配

### 5.2 色彩体系

| 用途 | 色值 | 说明 |
|------|------|------|
| 沉浸黑（顶栏/主功能区背景） | `#121214` | 电影灰黑，模拟影院沉浸感 |
| 深色背景-次级 | `#1A1A1E` | 卡片在深色区域的底色 |
| 聚光橙（品牌主色） | `#FF6B35` | 舞台聚光灯色，用于按钮、高亮标签、重要数值 |
| 聚光橙-浅底 | `rgba(255,107,53,0.1)` | 橙色标签的透明底色 |
| 内容底色 | `#F8F9FA` | 极浅灰，白色卡片区域的底色 |
| 卡片背景 | `#FFFFFF` | 纯白卡片，与内容底色形成微弱对比 |
| 玻璃材质背景 | `rgba(255,255,255,0.7)` | 配合 backdrop-blur 使用 |
| 辅助色-成功 | `#52C41A` | 通过、成功状态 |
| 辅助色-警告 | `#FAAD14` | 待审核、提醒状态 |
| 辅助色-危险 | `#FF4D4F` | 拒绝、错误状态 |
| 文字-深色区主文字 | `#FFFFFF` | 深色背景上的标题文字 |
| 文字-深色区辅助 | `rgba(255,255,255,0.6)` | 深色背景上的次要文字 |
| 文字-浅色区主文字 | `#1A1A1A` | 浅色卡片上的标题 |
| 文字-浅色区次要 | `#666666` | 浅色卡片上的正文 |
| 文字-浅色区辅助 | `#999999` | 提示文字、时间戳 |
| 分割线 | `#EEEEEE` | 列表分割线 |

### 5.3 视觉特征

**极致圆角 (Super Ellipse)：**
- 页面容器（如页头区域）：圆角 32px - 40px
- 卡片：圆角 24px
- 按钮：圆角 16px（常规）/ 48px（胶囊）
- 这种"超椭圆"设计让界面柔和且现代，消除专业工具的冰冷感

**负边距层级 (Negative Margin)：**
- 白色内容区域通过负边距向上方深色背景"叠放"
- 配合柔和长阴影，营造物理层级感
- 界面看起来像从深色影院背景中"长"出来的卡片
- 深浅色过渡区域使用 `border-radius: 32rpx 32rpx 0 0` 的圆角过渡

**玻璃拟态 (Glassmorphism)：**
- 底部悬浮导航栏：`backdrop-filter: blur(20px)` + 半透明背景
- 统计卡片/浮层：背景模糊 + 透明度组合
- 卡片悬浮感：微妙内发光边框 `border: 1rpx solid rgba(255,255,255,0.2)` + 外阴影
- 模拟在空气中悬浮的轻盈质感

### 5.4 字体层级 — 影视海报级对比

| 层级 | 字号 | 字重 | 用途 | 说明 |
|------|------|------|------|------|
| Display | 48rpx | Black (900) | 金额、核心数据 | 极粗字重，强视觉张力 |
| H1 | 40rpx | Bold (700) | 页面大标题 | 深色区域白色文字 |
| H2 | 34rpx | Bold (700) | 区块标题、角色名称 | |
| H3 | 30rpx | Medium (500) | 卡片标题 | |
| Body | 28rpx | Regular (400) | 正文内容 | |
| Caption | 24rpx | Regular (400) | 辅助说明、时间戳 | 较大字间距 |
| Mini | 22rpx | Regular (400) | 标签文字 | |

### 5.5 间距与布局

| 元素 | 数值 |
|------|------|
| 页面左右边距 | 32rpx |
| 深色头部区域高度 | 约 320rpx - 400rpx（含状态栏） |
| 白色内容区上移叠放 | 负边距 -40rpx |
| 卡片内边距 | 28rpx |
| 卡片间距 | 24rpx |
| 卡片圆角 | 24rpx |
| 输入框圆角 | 16rpx |
| 标签圆角 | 8rpx（方形标签）/ 全圆角（胶囊标签） |
| 头像圆角 | 50%（圆形） |

### 5.6 组件规范

**按钮：**
- 主按钮：聚光橙 `#FF6B35` 实心，白色文字，高度 96rpx，圆角 16rpx
- 次按钮：透明底 + 聚光橙边框 + 聚光橙文字
- 深色区按钮：`rgba(255,255,255,0.15)` 底 + 白色文字（玻璃质感）
- 危险按钮：透明底 + 红色边框 + 红色文字
- 禁用态：`rgba(255,255,255,0.1)` 底 + `rgba(255,255,255,0.3)` 文字

**卡片：**
- 白色背景，圆角 24rpx
- 阴影：`0 4rpx 24rpx rgba(0,0,0,0.06)`
- 内边距 28rpx
- 深色区域内的卡片：`rgba(255,255,255,0.08)` 底 + `border: 1rpx solid rgba(255,255,255,0.1)` + `backdrop-filter: blur(10px)`

**标签（Tag）：**
- 胶囊形状，全圆角
- 高度 44rpx，左右内边距 20rpx
- 浅色区：`#F5F5F5` 底 + 深色文字（保持画面整洁）
- 深色区：`rgba(255,107,53,0.15)` 底 + 橙色文字
- 状态标签：成功绿底/警告黄底/危险红底，均为低透明度

**输入框：**
- 高度 96rpx，圆角 16rpx
- 浅色区：`border: 1rpx solid #EEEEEE`，聚焦时边框变为聚光橙
- 深色区：`rgba(255,255,255,0.1)` 底，无边框，聚焦时底色加亮

**底部导航栏（TabBar）：**
- 玻璃拟态：`backdrop-filter: blur(20px)` + `background: rgba(255,255,255,0.85)`
- 悬浮于页面底部，带微妙上边框 `border-top: 1rpx solid rgba(0,0,0,0.05)`
- 选中态图标 + 文字为聚光橙，未选中为 `#999999`

**空状态：**
- 居中线性插画图标（白色/浅灰线条风格）
- 下方文字提示 + 聚光橙操作按钮

### 5.7 页面结构模式

每个页面遵循统一的「深-浅」双层结构：

```
┌──────────────────────────────┐
│  ██████████████████████████  │  ← 深色区域 #121214
│  ██  页面标题 / 核心信息  ██  │     顶部导航 + 页面标题 + 关键数据
│  ██████████████████████████  │
│  ╭────────────────────────╮  │  ← 白色内容区，圆角上移叠放
│  │                        │  │     border-radius: 32rpx 32rpx 0 0
│  │    主体内容区域          │  │     背景 #F8F9FA
│  │    （卡片列表/表单）     │  │
│  │                        │  │
│  │                        │  │
│  ╰────────────────────────╯  │
│  [  🏠首页  📋发现  👤我的  ] │  ← 玻璃拟态底部导航
└──────────────────────────────┘
```

### 5.8 图标风格

- 线性图标，线宽 2rpx
- 深色背景上使用白色图标
- 浅色背景上使用 `#666666` 图标
- 选中/激活态使用聚光橙 `#FF6B35`
- 推荐图标库：与 uview-plus 内置图标配合使用

## 十一、V1.1 新增功能 — 演员增强

> 新增日期：2026-03-23
> 三大功能：分享明信片、档案美化、信用积分体系

### 11.1 分享明信片（Actor Card）

**用途：** 演员可生成个人介绍明信片，分享到微信好友/朋友圈，他人点击可进入小程序查看演员详情。

**核心链路：**
```
演员「我的」→ 分享名片 → 明信片预览 → 分享/保存海报
                                        ↓
              好友/朋友圈点击 → 进入小程序 → 演员公开详情页
```

**明信片页面** `pages/actor-card/index`：

| 区域 | 内容 |
|------|------|
| 顶部 | 头像（大）+ 姓名 + 标签（性别/城市/擅长类型） |
| 数据栏 | 演员等级（LV.X）+ 积分徽章 + 完成拍摄次数 |
| 照片区 | 代表照片 3 张（从照片墙中选取） |
| 自我介绍 | 截取前 100 字 |
| 底部 | 小程序码 + "扫码查看完整档案" |

**交互行为：**
- 转发好友：`onShareAppMessage` 携带 `actorId` 参数
- 分享朋友圈：`onShareTimeline`
- 生成海报：Canvas 绘制图片 → 保存到相册
- 落地页：`actor-profile/detail` 改为公开可访问（不限身份）

**入口位置：**
- 「我的」页面新增"分享我的名片"按钮
- 演员资料编辑页底部新增"预览名片"按钮

### 11.2 档案美化（Profile Enhancement）

**用途：** 增强演员资料编辑页，让演员能打造更有吸引力的个人简历。

**在现有 `actor-profile/edit` 基础上新增：**

| 新增板块 | 说明 |
|---------|------|
| **作品经历** | 项目名、角色名、拍摄时间、剧照（演员简历最核心的缺失） |
| **形象标签扩展** | 在擅长类型基础上增加：体型、发型、语言/方言等 |
| **照片分类** | 照片墙分区：形象照、生活照、剧照（取代混合上传） |
| **自我介绍引导** | 提供模板/引导语帮助演员写出更好的自我介绍 |
| **档案评分优化** | 完整度计算更精细，给出具体"提升建议"（如"上传视频可提升20%"） |
| **预览名片** | 编辑页底部增加"预览我的名片"按钮，所见即所得 |

**作品经历数据结构：**
```
WorkExperience {
  projectName: string     // 项目名称
  roleName: string        // 角色名称
  shootDate: string       // 拍摄时间
  photos: string[]        // 剧照（最多 4 张）
  description?: string    // 简要描述
}
```

**新增类型定义** `types/actor.ts`：
- `WorkExperience` — 作品经历
- `ActorProfile` 扩展字段：`workExperiences`, `bodyType`, `hairStyle`, `languages`, `photoCategories`

### 11.3 信用积分与演员等级（Credit & Level）— 预留

**用途：** 单轨积分制。从 **0 分起步**，通过完善档案、完成拍摄、获得好评、保持活跃来累积积分（满分 100）。积分直接决定演员等级和徽章。

> 详细积分规则、数据结构、API 设计见 `.sce/specs/05-03-credit-score/requirements.md`

#### 积分来源（四大类，满分 100）

| 类别 | 满分 | 含义 |
|------|------|------|
| 档案建设 | 30 | 头像、基本信息、照片、视频、作品经历、形象标签 |
| 拍摄经验 | 30 | 完成拍摄里程碑（1次/5次/15次/30次/多类型） |
| 口碑评价 | 25 | 好评、好评率达标、连续无爽约 |
| 活跃贡献 | 15 | 投递、登录、分享名片 |

#### 扣分规则

| 行为 | 扣分 |
|------|------|
| 爽约未到场 | -10 |
| 迟到 | -5 |
| 获得差评 | -5 |
| 频繁取消已确认投递 | -3 |

最低扣至 0 分。

#### 等级阶梯（积分区间映射）

| 积分 | 等级 | 名称 | 徽章 |
|------|------|------|------|
| 0-9 | LV.1 | 新秀演员 | — |
| 10-24 | LV.2 | 潜力演员 | — |
| 25-44 | LV.3 | 活跃演员 | 🥉 |
| 45-59 | LV.4 | 认证演员 | 🥈 |
| 60-74 | LV.5 | 专业演员 | 🥈 |
| 75-89 | LV.6 | 明星演员 | 🥇 |
| 90-100 | LV.7 | 王牌演员 | 💎 |

#### 前期 mock 策略

mock 阶段仅"档案建设+部分活跃贡献"可得分，上限约 36 分（LV.3 活跃演员）。

#### 展示位置

| 位置 | 内容 |
|------|------|
| 「我的」页面头部 | LV.X + 名称标签 + 积分 + 徽章 |
| 「我的」页面菜单 | "我的信用分"入口 |
| 演员明信片 | 等级标签 + 积分徽章 |
| 演员详情落地页 | 等级标签 + 积分徽章 |
| 档案编辑页顶部 | 等级标签（替换硬编码 PRO ACTOR / LV.4） |
| 排行榜列表项 | 等级 + 积分 + 徽章 |

- `KpLevelTag` — 等级标签（LV.X + 名称，不同等级不同颜色）
