# 身份选择页

## 1. 概述

"开拍了"（KaiPai）小程序的身份选择页面，路由路径 `pages/role-select/index`。新用户登录后首次进入此页面，选择身份（演员或剧组）。身份一经选择不可更改，需二次确认。页面采用全深色沉浸式背景，两张大尺寸毛玻璃卡片垂直排列，选中态以品牌色橙光边框高亮。

## 2. 用户故事

- 作为**新注册用户**，我希望清晰地看到两种身份选项（演员/剧组），以便根据自己的需求做出选择。
- 作为**新用户**，我希望在确认身份前收到"不可更改"的提醒，避免误操作。
- 作为**选择了演员身份的用户**，我希望确认后直接进入演员资料编辑页，开始完善个人信息。
- 作为**选择了剧组身份的用户**，我希望确认后直接进入剧组资料编辑页，开始填写公司信息。

## 3. 功能需求

### 3.1 页面路由配置

**描述**: 在 `pages.json` 中注册身份选择页路由，配置为非 Tab 页面，导航栏样式为自定义（`navigationStyle: custom`），背景色 `#121214`。此页面仅在新用户首次登录后由登录页 `redirectTo` 进入，不可通过返回按钮回到登录页。

**验收标准**:
- WHEN 新用户登录成功且无角色 THEN 从登录页 `redirectTo` 到此页面
- WHEN 页面加载 THEN 系统导航栏隐藏，使用自定义全屏布局
- WHEN 用户尝试返回 THEN 无返回按钮，无法回退到登录页

### 3.2 身份卡片展示

**描述**: 页面垂直居中展示两张身份选择卡片（演员、剧组），每张卡片高度 280rpx，采用毛玻璃效果（`glass` 模式）。卡片内包含身份图标、身份名称、身份描述文字。两张卡片间距 32rpx。

**验收标准**:
- WHEN 页面加载 THEN 显示两张等高（280rpx）的毛玻璃卡片，垂直排列
- WHEN 卡片渲染 THEN 演员卡片显示演员图标 + "我是演员" + 描述文字
- WHEN 卡片渲染 THEN 剧组卡片显示剧组图标 + "我是剧组" + 描述文字
- WHEN 卡片未选中 THEN 呈半透明毛玻璃效果，边框为 `rgba(255,255,255,0.2)`
- WHEN 两张卡片 THEN 间距为 32rpx

### 3.3 卡片选中交互

**描述**: 用户点击卡片后，该卡片进入选中状态：边框变为品牌色 `#FF6B35`，外发橙色光晕（`box-shadow`），卡片内图标/文字高亮。同时另一张卡片恢复未选中状态（单选逻辑）。

**验收标准**:
- WHEN 用户点击演员卡片 THEN 演员卡片边框变为 `#FF6B35`，显示橙色外发光
- WHEN 用户点击剧组卡片 THEN 剧组卡片边框变为 `#FF6B35`，显示橙色外发光
- WHEN 选中一张卡片 THEN 另一张卡片恢复未选中样式
- WHEN 卡片选中 THEN 卡片内图标颜色变为 `#FF6B35`
- WHEN 选中状态切换 THEN 有 300ms 过渡动画

### 3.4 确认按钮

**描述**: 当用户选中任一身份卡片后，页面底部出现确认按钮（品牌色，大尺寸，块级）。未选中时按钮不显示或禁用。

**验收标准**:
- WHEN 未选中任何卡片 THEN 确认按钮不显示（或显示为禁用态）
- WHEN 选中任一卡片 THEN 确认按钮出现，文字为"确认选择"
- WHEN 确认按钮出现 THEN 有从底部滑入的动画效果

### 3.5 二次确认对话框

**描述**: 点击确认按钮后弹出 `KpConfirmDialog`，内容为"确定选择【演员/剧组】身份？选择后不可更改"，用户需再次点击确认才执行身份设置。

**验收标准**:
- WHEN 点击确认按钮 THEN 弹出 KpConfirmDialog 对话框
- WHEN 选择的是演员 THEN 对话框内容为"确定选择【演员】身份？选择后不可更改"
- WHEN 选择的是剧组 THEN 对话框内容为"确定选择【剧组】身份？选择后不可更改"
- WHEN 用户点击对话框"确认" THEN 调用 `updateUserRole(role)` API
- WHEN 用户点击对话框"取消" THEN 关闭对话框，保持当前选中状态
- WHEN API 调用中 THEN 确认按钮显示 loading，防止重复提交

### 3.6 身份确认后路由跳转

**描述**: 身份设置 API 成功后，更新本地用户信息（Pinia Store + Storage），然后跳转到对应的资料编辑页。

**验收标准**:
- WHEN 身份设置成功且选择演员 THEN 更新 `useUserStore` 中 role 为 1，跳转到 `/pages/actor-profile/edit`（使用 `redirectTo`）
- WHEN 身份设置成功且选择剧组 THEN 更新 `useUserStore` 中 role 为 2，跳转到 `/pages/company-profile/edit`（使用 `redirectTo`）
- WHEN 身份设置失败 THEN 提示错误信息，保持当前页面
- WHEN 路由跳转 THEN 使用 `redirectTo` 确保身份选择页不留在页面栈中

## 4. 非功能需求

> 通用非功能需求见 [SHARED_CONVENTIONS.md](../SHARED_CONVENTIONS.md)

- **性能**: 卡片选中动画流畅无卡顿（60fps）。
- **安全**: 身份设置 API 需携带 Token，防止未授权调用。
- **兼容性**: 毛玻璃效果在不支持 `backdrop-filter` 的机型上降级为纯色半透明背景。

## 5. 约束条件

> 通用约束见 [SHARED_CONVENTIONS.md](../SHARED_CONVENTIONS.md)

- 组件库：使用 `00-02-shared-components` 中的 KpCard（glass 模式）、KpButton、KpConfirmDialog
- API 层：使用 `00-03-shared-utils-api` 中的 `api/auth.ts` 的 `updateUserRole` 函数和 `stores/user.ts`
- 身份一经选择不可更改，此为产品层面的硬性约束
- 页面为非 Tab 页，无返回按钮，使用 `redirectTo` 进入和离开
