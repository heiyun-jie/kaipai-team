# 00-16 后台运营文案优化（Admin Operator Copy Optimization）

> 状态：已完成 | 优先级：P1 | 依赖：00-11 platform-admin-console
> 记录目的：把后台管理端中偏“接口接入/链路说明”的展示文案改为运营团队可直接理解的任务说明，减少页面阅读门槛。

## 1. 背景

当前后台多个页面仍保留了研发调试阶段的接口导向文案，例如“已接通某接口”“以某接口为准”等。这类信息对运营使用者帮助有限，也会干扰页面的业务理解。

## 2. 范围

### 2.1 本轮必须处理

- 顶部状态提示与通用页面说明
- 工作台概览、模块卡片、最近事项说明
- 实名认证等高频运营页的描述文案
- 登录页与占位页中的技术导向提示

### 2.2 本轮不处理

- 后端接口、权限、路由结构变更
- 页面视觉体系重做
- 新增多语言或文案配置中心

## 3. 需求

### 3.1 文案方向

- **R1** 页面文案必须优先描述“运营要做什么”，而不是“前后端接了什么接口”
- **R2** 非研发必需的信息中，不再直接展示接口路径、DTO、聚合接口等技术描述
- **R3** 同一页面内的标题、说明、状态标签语气必须统一，避免一部分像后台任务，一部分像联调备注

### 3.2 工作台

- **R4** 工作台标题区、概览卡片、模块卡片必须改成运营视角说明
- **R5** 模块状态标签要表达可处理状态，如“待处理 / 运行中 / 可配置”，而不是“已接接口”
- **R6** 最近事项说明要突出待办与回看价值，而不是后端聚合来源

### 3.3 通用后台页

- **R7** 登录页、占位页、顶部提示要突出后台用途、任务边界和当前状态
- **R8** 高频业务页描述要说明页面能完成的运营动作
