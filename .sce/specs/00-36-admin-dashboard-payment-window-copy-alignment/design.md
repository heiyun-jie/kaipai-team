# 00-36 设计说明

## 1. 设计原则

- 不碰统计逻辑，只收口展示语义
- 未设置时间窗口时保留“今日”语义
- 一旦进入自定义时间窗口，所有支付展示都改为时间窗口口径

## 2. 设计策略

### 2.1 支付统计卡

通过当前筛查状态动态生成支付卡标题与 hint：

- 无 `dateFrom/dateTo`：
  - 标题：`今日支付订单`
  - hint：`按今日业务数据统计的支付订单数量`
- 有 `dateFrom/dateTo`：
  - 标题：`时间窗口支付订单`
  - hint：`按当前时间窗口统计的支付订单数量`

### 2.2 支付最近事项标题

为最近事项标题增加前端展示层转换：

- 非支付事项继续显示原始标题
- 支付事项在无时间窗口时显示 `今日支付订单`
- 支付事项在有时间窗口时显示 `支付订单记录`
