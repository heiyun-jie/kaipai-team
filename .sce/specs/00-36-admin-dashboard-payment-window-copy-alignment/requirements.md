# 00-36 后台工作台支付时间窗口文案对齐（Admin Dashboard Payment Window Copy Alignment）

> 状态：进行中 | 优先级：P1 | 依赖：00-35 admin-dashboard-filter-scope-clarity
> 记录目的：修正工作台支付相关卡片与最近事项在自定义时间窗口下仍显示“今日”造成的口径误导。

## 1. 背景

后端 `AdminDashboardServiceImpl` 当前行为为：

- 未传 `dateFrom/dateTo` 时，支付统计与支付最近事项默认取今日窗口。
- 传入 `dateFrom/dateTo` 时，支付统计与支付最近事项改按当前时间窗口统计。

但当前前端页面仍固定展示：

- 统计卡：`今日支付订单`
- 提示文案：`按今日业务数据统计的支付订单数量`
- 支付最近事项标题也会继续显示 `今日支付订单`

这会让运营在查看历史窗口或自定义时间窗口时误以为支付数据仍是“今日”口径。

## 2. 范围

### 2.1 本轮必须处理

- `kaipai-admin/src/views/dashboard/OverviewView.vue`

### 2.2 本轮不处理

- dashboard 后端统计逻辑
- DTO 字段名重命名
- 其他业务线文案调整

## 3. 需求

### 3.1 统计卡

- **R1** 未设置时间窗口时，支付统计卡可以继续展示“今日支付订单”。
- **R2** 已设置时间窗口时，支付统计卡不得继续固定展示“今日支付订单”。
- **R3** 已设置时间窗口时，支付统计卡提示文案必须同步切换到“当前时间窗口”口径。

### 3.2 最近事项

- **R4** 已设置时间窗口时，支付类最近事项标题不得继续使用“今日支付订单”。
- **R5** 最近事项支付标题应与当前筛查口径保持一致。

### 3.3 边界

- **R6** 只调整展示语义，不得改动 payment 统计逻辑。
- **R7** 展示语义应根据当前时间窗口实时更新。
