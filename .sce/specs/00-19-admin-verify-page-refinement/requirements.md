# 00-19 实名认证审核页二次优化（Admin Verify Page Refinement）

> 状态：已完成 | 优先级：P1 | 依赖：00-18 admin-page-style-alignment
> 记录目的：解决实名认证审核页输入区可见性差、文字垂直对齐不稳和页面层级仍偏平的问题。

## 1. 背景

实名认证待审核页虽然已经接入统一后台壳层，但当前筛选区仍保留行内表单结构：标签和输入控件挤在一排，输入背景与页面底色过近，导致输入框边界和占位内容不够清楚，页面重心也不稳定。

## 2. 范围

### 2.1 本轮必须处理

- 实名认证待审核 / 历史页的筛选区结构与视觉
- 输入框、下拉框的对比度与垂直对齐
- 页内概览提示、表格空态和详情展示的视觉收敛
- 当前页操作区和状态区的可读性强化

### 2.2 本轮不处理

- 后端接口、字段和审核流程
- 后台其他业务页的逐页重做
- 侧边栏、顶栏和 dashboard 的再次改版

## 3. 需求

### 3.1 筛选区

- **R1** 标签与输入控件必须拉开层级，避免同一行挤压造成输入框不清晰
- **R2** 输入框、下拉框和占位文字对比度必须明显高于页面底色
- **R3** 输入框和下拉框中的文字、图标在容器内需要保持稳定的垂直居中

### 3.2 页面层级

- **R4** 当前页需要有清晰的审核说明和状态概览，而不是只剩标题和表格
- **R5** 表格空态需要体现“当前无待处理数据”的运营语义，而不是默认空表
- **R6** 详情抽屉中的信息块和动作说明要与当前页主视觉一致

- [x] 实名认证审核页输入区已解决可见性弱和垂直对齐问题
- [x] 当前页的概览、筛选、空态和详情展示已完成页面级收敛
