# 05-05 名片分享与等级体系 v2（Card Share & Level System）

> 状态：架构重设计 | 优先级：最高 | 依赖：05-09（实名认证）、05-10（邀请裂变）、05-08（命理���性化）
> 替代关系：本 Spec v2 替换 v1 的 basic/pro 二元会员模型，改为邀请驱动多级制 + 场景名片

## 1. 功能概述

产品主线从"名片 + 二元会员"升级为"名片 + 邀请等级 + 场景定制 + 命理个性化"。

核心变化：

1. 会员模型从 `basic/pro` 改为**邀请人数驱动的 6 级体系**
2. 名片从固定 3 套模板改为**场景名片**（古装/都市/商业/文艺等）+ 可扩展模板
3. AI 简历润色对普通用户开放（限次），不再是会员专属
4. 新增命理个性化���统（幸运色/生肖/星座/紫微斗数），与名片定制联动
5. 模板初期免费，后期部分转为付费皮肤

## 3. 需求清单

### 3.1 等级体系

- **R1** 用户等级由邀请人数驱动���共 6 级（Lv0-Lv5）
- **R2** 等级解锁前置条件：实名认证通过 + 档案完成度 ≥ 70%
- **R3** 等级定义：

| 等级 | 邀请人数 | 解锁能力 |
|------|---------|---------|
| Lv0 未认证 | — | 浏览通告、投递，不能生成名片 |
| Lv1 已认证 | 0 | 基础场景模板 + 基础分享 + AI 简历(1次/月, 新用户额外+2) |
| Lv2 | ≥ 1 | 解锁更多场景模板 |
| Lv3 | ≥ 3 | 解锁配色微调 + AI(3次/月) |
| Lv4 | ≥ 5 | 解锁布局选择 + 全部场景模板 |
| Lv5 | ≥ 8 | 全定制能力 + AI(5次/月) + 命理配色 + 未来付费皮肤免费体验 |

- **R4** 等级只升不降（邀请人注销不影响已达等级）
- **R5** 等级页面展示当前等级、邀请进度、下一级所需邀请数、已解锁能力清单

### 3.2 场景名片

- **R6** 名片支持按面试场景切换，每个场景对应独立的视觉风格和内容侧重
- **R7** ���始场景类型：

| 场景 Key | 名称 | 视觉风格 | 内容侧重 |
|---------|------|---------|---------|
| `costume` | 古装 | 水墨/国风配色，传统纹理 | 古装剧经历、古装照、身段/武戏标签 |
| `urban` | 都市 | 现代简约，冷色调 | 都市剧经历、生活照、台词/情感表达标签 |
| `commercial` | 商业 | 明亮活力，高饱和 | 广告/短视频经历、形象照、镜头感标签 |
| `artistic` | 文艺 | 胶片质感，低饱和 | 独立电影/文艺片经历、意境照、表演深度标签 |
| `general` | 通用 | 当前 starter 风格 | 全量档案信息，不做偏重 |

- **R8** 演员可为每个场景独立配置：突出的经历、展示的照片、标签排序
- **R9** 分���时场景参数 `scene` 写入分享链接，访客看到的是对应场景风格的名片
- **R10** Lv1 可用 `general` + 1 个场景；Lv2 解锁 3 个；Lv4+ 全部解锁
- **R11** 场景模板配置由后端接口下发，前端不硬编码模板资源

### 3.3 名片主页面

- **R12** 名片页区分"本人预览态"和"分享访客态"
- **R13** 本人预览态可切换场景、定制配色（按等级）、触发 AI 润色
- **R14** 分享访客态直接展示最终名片结果，不暴露管理控件
- **R15** 名片页必须展示：头像、姓名、核心标签、场景相关经历、场景相关照片、档案完成度
- **R16** 页面支持返回逻辑兜底：有栈返回，无栈时回首页或"我的"

### 3.4 分享模型

- **R17** 小程序分享路径统一使用 `pkg-card/actor-card/index`
- **R18** 分享参数：`actorId`、`scene`、`tone`、`shared`
- **R19** 分享进入后，页面必须按参数恢复场景与文案风格
- **R20** 分享卡片标题和海报文案跟随场景变化

### 3.5 配色定制

- **R21** 定制分三层，按等级递进解锁：

| 层级 | 等级要求 | 能力 |
|------|---------|------|
| L1 预设模板 | Lv1+ | 选择预设场景模板，不可改配色 |
| L2 配色微调 | Lv3+ | 在预设基础上修改主色/强调色/背景色 |
| L3 布局选择 | Lv4+ | 2-3 种布局变体（紧凑/宽松/杂志风） |

- **R22** 配色定制通过色盘选择器实现，不做低代码拖拽编辑
- **R23** 用户定制配置存储在后端，分享访客通过接口获取该演员的定制配置
- **R24** 命理系统（05-08）产出的幸运色可一键应用为名片主色

### 3.6 海报能力

- **R25** 名片页支持生成海报并保存到相册
- **R26** 海报风格跟随当前场景模板
- **R27** 海报生成失败不影响页面继续分享或浏览

### 3.7 AI 简历润色

- **R28** AI 简历润色对所有已认证用户开放（不再是会员专属）
- **R29** 次数限制：

| 用户类型 | 每月次数 |
|---------|---------|
| 新用户（注册首月） | 3 次（1 基础 + 2 赠送） |
| Lv1-Lv2 | 1 次/月 |
| Lv3 | 3 次/月 |
| Lv4 | 4 次/月 |
| Lv5 | 5 次/月 |

- **R30** AI 润色范围：标题、名片介绍、场景说明、分享说明
- **R31** 当前允许前端 mock / 本地规则生成，后续对接真实大模型

### 3.8 公开详情页

- **R32** `pages/actor-profile/detail` 保持公开可访问
- **R33** 详情页展示：自我介绍、拍摄经历、照片墙、视频简历、联系方式入口
- **R34** 详情页提供"查看名片"入口，跳回对应演员名片页

### 3.9 我的页入口

- **R35** "我的"页提供"我的名片"、"我的等级"入口
- **R36** 当前主线中不得出现"我的信用分""排行榜"等旧入口

### 3.10 付费皮肤预埋

- **R37** 模板数据模型预留 `tier` 字段：`free` / `paid`
- **R38** 初期所有模板 `tier = free`
- **R39** 后期付费皮肤需支付后解锁，Lv5 用户可免费体验

### 3.11 清理约束

- **R40** 删除旧 `basic/pro` 会员类型定义、旧会员能力矩阵、旧会员页逻辑
- **R41** 删除旧 `MembershipPlan` 枚举和相关判断代码
- **R42** 保留旧 `audience`（crew/brand/friend）作为分享受众维度，与场景 `scene` 正交

## 4. 数据模型

### 4.1 用户等级

```ts
enum UserLevel {
  Lv0 = 0, // 未认证
  Lv1 = 1, // 已认证
  Lv2 = 2, // 邀请 1 人
  Lv3 = 3, // 邀请 3 人
  Lv4 = 4, // 邀请 5 人
  Lv5 = 5, // 邀请 8 人
}

interface UserLevelInfo {
  level: UserLevel
  inviteCount: number
  nextLevelRequirement: number | null
  isCertified: boolean
  profileCompletion: number
}
```

### 4.2 场景名片

```ts
type CardScene = 'general' | 'costume' | 'urban' | 'commercial' | 'artistic'

interface SceneTemplate {
  sceneKey: CardScene
  name: string
  description: string
  themeColors: { primary: string; accent: string; background: string }
  layoutVariant: 'compact' | 'spacious' | 'magazine'
  contentFocus: string[] // 突出的内容板块
  tier: 'free' | 'paid'
  requiredLevel: UserLevel
}
```

### 4.3 名片定制配置

```ts
interface ActorCardConfig {
  actorId: number
  sceneKey: CardScene
  layoutVariant: 'compact' | 'spacious' | 'magazine'
  primaryColor: string   // hex
  accentColor: string    // hex
  backgroundColor: string // hex
  highlightedExperiences: number[] // 突出的经历 ID
  highlightedPhotos: string[]     // 突出的照片 URL
  tagOrder: string[]              // 标签排序
}
```

### 4.4 分享参数

```ts
interface ActorCardShareParams {
  actorId: number
  scene: CardScene
  tone: 'natural' | 'professional' | 'commercial'
  shared: '1'
}
```

### 4.5 AI 使用配额

interface AiUsageQuota {

  usedCount: number
