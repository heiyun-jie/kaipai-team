# 当前主线残余清理 - 技术设计

_Requirements: 05-06 全部_

## 1. 清理边界

本轮不做新的业务能力开发，只做当前主线的表达收敛与残余删除。

允许修改：

- `pages/mine/index` 的身份区和统计区
- `pages/actor-profile/edit` 的档案状态文案
- 必要的计算属性与辅助函数

不修改：

- 分享参数模型
- 会员能力矩阵
- 信用体系历史 Spec

## 2. 清理策略

### 2.1 身份表达拆维

当前主线至少存在三个维度：

- 账号角色：演员 / 剧组
- 档案状态：待完善 / 已成型 / 可分享
- 会员状态：基础版 / 会员版

实现上应分别展示，不再用 `PRO ACTOR` 这类旧英文口号混写。

### 2.2 “我的”页统计收敛

“我的”页统计只保留现有可算字段，例如：

- 拍摄经历数
- 可用模板数
- 投递中数量

不再展示无法从当前仓库状态得到的数据，例如虚构浏览量。

### 2.3 编辑页状态文案

`actor-profile/edit` 顶部状态改为与档案完成度直接相关的中文档案状态，不再使用英文等级式文案。

建议口径：

- `<60`: `档案整理中`
- `60-84`: `资料已成型`
- `>=85`: `可分享名片`

## 3. 验证

- 文本扫描：`PRO ACTOR / ACTIVE ACTOR / ACTOR / TEAM`
- 文本扫描：`credit / rank / 积分 / 排行榜`
- `npm run type-check`
