# 00-23 后台固定列层级防穿透修复（Admin Fixed Column Layer Fix）

> 状态：已完成 | 优先级：P1 | 依赖：00-22 admin-table-action-overflow-fix
> 记录目的：修复后台表格 fixed 右侧列背景透明导致底层普通列内容透入的问题，保证操作区边界稳定。

## 1. 背景

`00-22` 已把后台高风险操作列收口为统一按钮组容器，但当前后台全局表格样式将 `tr` 背景设为透明，`fixed="right"` 的单元格没有单独补不透明背景和边界层。结果是在后台账号页这类多列宽表格中，普通列文本会从 fixed 操作列底部透上来，视觉上仍像“操作按钮被列表覆盖”。

## 2. 范围

### 2.1 本轮必须处理

- 后台 `table-card` 下 `el-table` fixed 右侧列的背景、边界和层级样式
- 表头与表体 fixed 列在普通态、hover 态下的可读性
- 保持已有 `.table-actions` 容器和页面业务逻辑不变

### 2.2 本轮不处理

- 表格字段顺序和业务动作调整
- 非 fixed 列的整体视觉重做
- 弹窗、抽屉和详情区样式改版

## 3. 需求

### 3.1 fixed 列稳定性

- **R1** 后台 fixed 右侧列必须拥有独立的不透明背景，不能让普通列文本透入
- **R2** fixed 列表头和表体需要有清晰的左侧分隔边界，便于识别冻结区域
- **R3** 行 hover 时，fixed 列需要与整行 hover 态保持一致，不出现层级断裂

### 3.2 收敛要求

- **R4** 本轮修复应优先落在后台共享样式，不在每个页面重复打补丁
- **R5** `AdminUsersView` 以及其他已接入 `.table-actions` 的 fixed 操作列页面都应自动受益

- [x] 后台 fixed 右侧列已不再透出底层普通列内容
- [x] 后台账号页操作区与相邻列边界清晰
