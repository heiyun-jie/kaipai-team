# 登录鉴权与前台会话闭环后台执行卡

## 1. 执行卡名称

登录鉴权与前台会话闭环 - 后台 / 运行配置执行卡

## 2. 归属切片

- `../../slices/login-auth-capability-slice.md`

## 3. 负责范围

- 小程序登录运行时配置台账
- 微信登录开关与环境变量盘点
- 联调账号、联调环境与阻塞项登记
- 运行时错误口径和收口结论

## 4. 不负责范围

- 登录页视觉实现
- 后端鉴权代码实现
- 后台账号登录本身
- 独立配置中心建设

## 5. 关键输入

- 上位 Spec：
  - `00-10 platform-admin-backend-architecture`
  - `00-11 platform-admin-console`
  - `01-01 page-login`
  - `00-28 architecture-driven-delivery-governance`
- 关键文件：
  - `kaipaile-server/src/main/resources/application.yml`
  - `kaipai-frontend/src/utils/runtime.ts`
  - `kaipai-frontend/src/pages/login/index.vue`
  - `00-28` 下本切片状态卡与联调卡

## 6. 目标交付物

- 一份明确的登录运行时配置清单
- 一份明确的联调前置条件清单
- 一份明确的真实环境阻塞记录
- 一条“不配就不启用”的治理口径

## 7. 关键任务

1. 盘点前端运行时
   - `VITE_API_BASE_URL`
   - `VITE_USE_MOCK`
   - `VITE_ENABLE_WECHAT_AUTH`
2. 盘点后端运行时
   - `WECHAT_MINIAPP_APP_ID`
   - `WECHAT_MINIAPP_APP_SECRET`
3. 固化启用规则
   - 前端显式启用且后端配置齐全，才允许把微信按钮当作真能力
   - 任一项缺失，都只能记为外部依赖阻塞
4. 固化联调记录
   - 联调环境
   - 测试账号
   - 报错截图 / 报文
   - 回填日期与结论

## 8. 依赖项

- 运维或环境维护方必须提供真实配置
- 前后端都要接受“配置不齐就不算完成”的治理口径
- 联调证据要能回填到 `00-28`，不能只停留在口头确认

## 9. 验证方式

- 能明确列出当前环境是否开启微信登录
- 能明确列出阻塞是在前端开关、后端配置还是微信侧
- 登录页提示与配置台账结论一致
- 联调记录可回填到状态卡

## 10. 完成定义

- 运行时配置台账完整
- 微信登录启用条件明确
- 配置缺失时阻塞原因可追踪
- 后续联调可按同一份准入条件执行

## 11. 风险与备注

- 若只改前端开关或只改后端配置其一，不算完成启用
- 若没有联调台账，最容易把“某个同学本地测通过”误写成“环境已接通”
- 本卡当前不新增独立后台页面，重点是把运行治理显式化
