# 剧组档案、项目、角色与投递最小连通闭环联调执行卡

## 1. 执行卡名称

剧组档案、项目、角色与投递最小连通闭环 - 联调执行卡

## 2. 归属切片

- `../../slices/crew-company-project-recruit-capability-slice.md`

## 3. 负责范围

- 真实环境账号、运行时、接口、后台动作的收口验证
- 发现并记录状态错口径、权限问题、数据兼容问题
- 回填状态文档

## 4. 不负责范围

- 补做缺失功能
- 设计新字段或重构项目模型
- 代替后台或前端做开发实现

## 5. 关键输入

- `status/crew-company-project-status.md`
- `status/recruit-role-apply-status.md`
- 本目录 3 张执行卡

## 6. 目标交付物

- 一次真实环境 `company -> project -> role -> apply-manage` 验证记录
- 一次后台项目 / 角色状态治理验证记录
- 明确的阻塞项和遗留风险

## 7. 关键任务

1. 用剧组账号保存档案并创建项目、角色
2. 用演员账号完成搜索、查看详情、投递、查看我的投递
3. 在后台结束项目，确认关联角色同步收口
4. 在后台暂停 / 恢复 / 结束角色，确认演员端结果同步变化
5. 回填状态页和下一轮最小动作

## 8. 依赖项

- 真实后端、真实小程序、真实后台都可访问
- 后台账号当前至少具备 `page.system.admin-users` fallback
- 需要准备一条存在关联关系的项目和角色样本

## 9. 验证方式

- 记录请求和页面结果，不只看接口 200
- 验证项目结束后演员端不再看到对应招募角色
- 验证项目恢复后不会自动恢复角色状态
- 验证角色恢复招募时若项目已结束会被正确拦截

## 10. 完成定义

- 三端都跑过一次真实链路
- 状态联动规则已被真实验证
- 状态文档已回填当前结论和阻塞项

## 11. 风险与备注

- 只跑 happy path 不足以证明治理动作真实可用
- 若缺少真实管理员权限，按钮展示可能正常但接口仍无法验证
- 若历史数据存在无项目关联角色，需要单独记录兼容性样本
