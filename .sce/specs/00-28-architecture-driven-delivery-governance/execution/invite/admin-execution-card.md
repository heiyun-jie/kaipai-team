# 邀请裂变与邀请资格闭环后台执行卡

## 1. 执行卡名称

邀请裂变与邀请资格闭环 - 后台管理端执行卡

## 2. 归属切片

- `../../slices/invite-referral-capability-slice.md`

## 3. 负责范围

- 邀请记录页、异常邀请复核页、邀请规则配置页、邀请资格管理页
- 页面筛选、详情抽屉、操作弹窗、状态标签和错误处理
- 后台路由、菜单、权限点与治理动作对齐
- 运营视角的记录查询、规则调整、资格发放与回看入口

## 4. 不负责范围

- `referral_record` 生效规则、注册绑定和资格落库逻辑
- 小程序端邀请页、登录页和分享卡片
- `admin_operation_log` 的后端落库实现
- 任何脱离邀请闭环的营销活动页扩展

## 6. 目标交付物

- 现有异常邀请复核页可稳定处理风险记录
- 补齐邀请记录页、规则配置页、资格管理页，与复核页组成完整治理入口
- 页面统一消费后台聚合接口，不自行拼多套前台接口
- 权限、菜单、路由和按钮动作一一对齐
- 后台能给出“记录可查、风险可审、规则可改、资格可发”的完整运营闭环

## 7. 关键任务

1. 固化现有异常邀请复核页
   - `RiskView.vue` 继续承担风险记录查询、详情查看和通过 / 作废 / 复核完成动作
   - 风险原因、设备命中、同小时命中、历史日志等字段完整回显
2. 补齐邀请记录页
   - 展示邀请人、被邀请人、邀请码、状态、风险标记、注册时间、生效时间
   - 记录页聚焦查询和回看，不承担规则修改
3. 补齐邀请规则配置页
   - 查看规则列表、详情、启用 / 停用、创建 / 编辑
   - 清楚区分实名前置、资料门槛、设备限制、频率限制等规则项
4. 补齐邀请资格管理页
   - 查看资格列表与详情
   - 支持手工发放、撤销、延期
   - 明确资格动作与邀请事实、会员能力之间的关系说明
5. 对齐后台治理入口
   - 路由、菜单、权限码、按钮显隐同步更新
   - 不要只保留 `referral/risk` 单页而让记录、策略、资格继续缺位

## 8. 依赖项

- 后端需要先稳定记录、规则、资格三组接口和对应 DTO
- 权限码必须与服务端 `@PreAuthorize` 保持一致
- 若要展示资格与邀请统计摘要，需要后端提供聚合字段而不是由后台页面自行推断

## 9. 验证方式

- 风险页可筛选、查看详情并执行通过 / 作废 / 复核完成
- 邀请记录页可按邀请码、邀请人、被邀请人、状态筛选并查看详情
- 规则配置页可创建、修改、启停规则且刷新后结果一致
- 资格管理页可发放、撤销、延期并查看历史状态
- 无权限账号看不到对应页面或动作按钮，接口访问被正确拦截

## 10. 完成定义

- 后台不再只有异常邀请单页，而是形成完整邀请治理入口
- 记录、风险、规则、资格四类页面可真实服务运营
- 页面状态、详情字段、权限与动作口径一致
- 后台消费的是统一后端契约，而不是页面自行拼口径
- 可以支撑邀请闭环的联调验收和后续日常运营

## 11. 风险与备注

- 当前 `kaipai-admin/src/api/referral.ts` 与相关页面只覆盖风险复核，记录、规则、资格管理页仍然缺口明显
- 若菜单和权限只补页面不补动作，最终会出现“能看不能改”或“按钮可见但接口 403”的治理假闭环
- 若后台页面自行推断资格状态而不消费后端事实字段，会再次违背统一口径原则
