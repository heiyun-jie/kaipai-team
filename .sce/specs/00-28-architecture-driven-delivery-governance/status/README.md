# 00-28 状态回填

本目录用于承接 `00-28` 的 `T13`：每轮实现完成后，回填该能力切片的联调结论和验收状态。

## 目标

- 让 `00-28` 不只停留在“切片卡 / 执行卡”，还能持续沉淀真实推进状态
- 每个能力切片都保留一份“当前真实状态”文档，避免把页面完成误判成闭环完成
- 为后续多线程推进提供统一收口入口

## 使用方式

1. 每个能力切片对应一份状态文档
2. 每轮实现完成后，先更新“当前判定 / 联调结论 / 验收判断 / 下一轮最小动作”
3. 再追加一条带日期的“回填记录”
4. 若六条闭环条件未同时满足，不得写成“闭环完成”

## 当前文档

- `verify-status.md`
- `invite-status.md`
- `membership-status.md`
- `ai-resume-status.md`
- `_status-template.md`

## 判定口径

- `闭环完成`：满足 `design.md` 第 7.2 节全部条件
- `局部完成`：已有部分前端 / 后端 / 后台能力，但端到端仍未闭合
- `演示态`：只能展示局部效果或 mock，不具备真实业务闭环能力
