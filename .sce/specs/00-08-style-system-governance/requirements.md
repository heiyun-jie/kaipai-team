# 00-08 样式体系减负和升级治理（Style System Governance）

> 状态：已完成（第一阶段） | 优先级：高 | 依赖：00-01, 00-04

## 1. 目标

对当前前端样式体系进行减负和升级治理，先解决全局 `@import` 弃用链路，收敛样式入口与注入方式，为后续继续治理 Sass 依赖和样式复杂度提供稳定基础。

## 2. 本轮范围

- 样式全局注入策略升级
- `@import` 到 `@use/@forward` 的首批迁移
- 样式入口模块收敛
- 构建告警变化验证

## 3. 需求

### 3.1 样式入口治理

- **R1** 当前样式体系必须收敛到明确的全局注入入口，不得继续在多个位置重复维护 `@import` 链。
- **R2** `vite.config.ts` 中的全局样式注入必须从 `@import` 升级为 `@use/@forward` 方案。
- **R3** `App.vue` 与 `src/styles/index.scss` 中的样式入口必须同步升级，不得保留旧 `@import`。

### 3.2 Sass 模块化

- **R4** `tokens / mixins / reset / page-layout` 必须形成清晰的依赖层次，避免隐式全局污染。
- **R5** 需要新增明确的样式注入桥接层，用于向页面和组件继续提供兼容的变量 / mixin 能力。
- **R6** 本轮不强行重写页面样式写法，现有页面和组件继续允许直接使用既有 `$kp-*` 变量和 `@include`，但底层注入方式必须升级。

### 3.3 验证

- **R7** 必须执行 `npm run type-check`
- **R8** 必须执行 `npm run build:mp-weixin`
- **R9** 必须对比构建日志，确认 `@import` 相关弃用告警显著收敛或消失。

### 3.4 边界

- **R10** 本轮不处理 Dart Sass `legacy-js-api` 告警本身，除非能在不引入高风险升级的前提下顺手解决。
- **R11** 本轮不大规模改写单页视觉样式，不新增业务功能。

## 4. 验收标准

- [x] 已建立新的样式注入桥接层
- [x] 已将全局注入与样式入口从 `@import` 迁移到 `@use/@forward`
- [x] `npm run type-check` 通过
- [x] `npm run build:mp-weixin` 通过
- [x] `@import` 相关 Sass 弃用告警已消失或显著收敛
