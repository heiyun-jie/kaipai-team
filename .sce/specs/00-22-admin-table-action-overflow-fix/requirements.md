# 00-22 后台账号表格操作列防穿透修复（Admin Table Action Overflow Fix）

> 状态：已完成 | 优先级：P1 | 依赖：00-18 admin-page-style-alignment
> 记录目的：修复后台账号页操作列按钮在 fixed 列中穿透覆盖相邻列的问题，保证运营操作区清晰可读。

## 1. 背景

后台账号页当前在操作列中同时放置查看详情、编辑、绑定角色、重置密码和启停用等多个动作。操作列为右侧 fixed 列，但按钮直接平铺输出，宽度不足时会挤压并覆盖到相邻列，运营同学难以分辨操作内容和数据边界。

## 2. 范围

### 2.1 本轮必须处理

- `AdminUsersView` 表格操作列的布局与换行约束
- fixed 操作列在窄宽度下的可读性和边界稳定性
- 保持当前动作集合、权限控制和交互行为不变

### 2.2 本轮不处理

- 其他后台页面的表格二次改版
- 后台账号的业务字段、接口和权限逻辑调整
- 全局表格组件重构

## 3. 需求

### 3.1 布局要求

- **R1** 后台账号页操作列中的按钮必须收纳在统一容器内，避免直接平铺导致内容溢出
- **R2** fixed 操作列在常见后台宽度下不得覆盖相邻列内容，按钮可换行但不允许文字互相压叠
- **R3** 操作按钮需要保持清晰的行列关系，便于运营快速识别“查看 / 编辑 / 绑定 / 重置 / 启停用”

### 3.2 稳定性要求

- **R4** 本轮调整不得改变原有按钮权限控制、点击行为和接口调用
- **R5** 页面仍需保持与当前后台共享风格一致，不引入新的视觉体系
