# 登录页

## 1. 概述

"开拍了"（KaiPai）小程序的登录页面，路由路径 `pages/login/index`。支持手机号 + 短信验证码登录，新用户自动注册。同时提供微信一键登录能力。页面采用全深色沉浸式背景（`#121214`），Logo 居中置顶，半透明输入区域，品牌色登录按钮，营造影视行业专业质感。

## 2. 用户故事

- 作为**新用户**，我希望通过手机号快速注册并登录，无需单独的注册流程，以便尽快开始使用。
- 作为**老用户**，我希望输入手机号和验证码即可登录，流程简洁高效。
- 作为**微信用户**，我希望通过微信一键授权获取手机号登录，免去手动输入的麻烦。
- 作为**用户**，我希望在登录前必须同意用户协议和隐私政策，以了解我的权益。
- 作为**已选择身份的用户**，我希望登录后直接进入首页；作为**未选择身份的新用户**，我希望登录后被引导至身份选择页。

## 3. 功能需求

### 3.1 页面路由配置

**描述**: 在 `pages.json` 中注册登录页路由，配置为非 Tab 页面，导航栏样式为自定义（`navigationStyle: custom`），背景色 `#121214`。

**验收标准**:
- WHEN 应用启动且用户未登录 THEN 跳转到 `/pages/login/index`
- WHEN 页面加载 THEN 系统导航栏隐藏，使用自定义全屏布局
- WHEN 页面背景 THEN 显示为纯深色 `#121214`

### 3.2 手机号输入

**描述**: 提供手机号输入框，限制为 11 位数字，带手机图标前缀，半透明深色输入框样式（`rgba(255,255,255,0.08)` 背景 + 白色文字）。实时校验格式，非法时显示错误提示。

**验收标准**:
- WHEN 用户输入手机号 THEN 仅允许输入数字，最大长度 11 位
- WHEN 手机号不满 11 位 THEN "获取验证码"按钮和"登录"按钮保持禁用
- WHEN 手机号格式不合法（不以 1 开头或不足 11 位） THEN 点击获取验证码时提示"请输入正确的手机号"
- WHEN 输入框获得焦点 THEN 边框变为品牌色 `#FF6B35` 半透明描边
- WHEN 输入框有内容且获得焦点 THEN 显示清除按钮

### 3.3 验证码获取与倒计时

**描述**: 验证码输入框右侧内嵌"获取验证码"文字按钮。点击后调用发送验证码 API，成功后按钮进入 60 秒倒计时状态，倒计时期间按钮禁用并显示剩余秒数。验证码为 6 位数字。

**验收标准**:
- WHEN 手机号合法且点击"获取验证码" THEN 调用 `sendSmsCode(phone)` API
- WHEN API 调用成功 THEN 按钮文字变为"60s 后重新获取"并开始倒计时
- WHEN 倒计时进行中 THEN 按钮禁用，文字每秒更新（59s、58s...1s）
- WHEN 倒计时结束 THEN 按钮恢复为"获取验证码"可点击状态
- WHEN 验证码输入 THEN 仅允许数字，最大长度 6 位
- WHEN API 调用失败 THEN 提示错误信息，按钮保持可点击

### 3.4 登录/注册逻辑

**描述**: 点击登录按钮，调用手机号登录 API。后端自动判断：已注册用户直接登录，未注册用户自动创建账号并登录。登录成功后将 Token 和用户信息持久化到本地存储和 Pinia Store。

**验收标准**:
- WHEN 手机号 11 位且验证码 6 位且已勾选协议 THEN 登录按钮启用
- WHEN 点击登录按钮 THEN 按钮显示 loading 状态，调用 `loginByPhone(phone, code)` API
- WHEN 登录成功 THEN Token 和 UserInfo 存入 `useUserStore` 和 `uni.setStorageSync`
- WHEN 登录失败 THEN 提示错误信息（如"验证码错误"），按钮恢复可点击
- WHEN 登录过程中 THEN 禁止重复点击（防抖）

### 3.5 微信一键登录

**描述**: 提供微信一键登录按钮（毛玻璃样式），使用微信 `getPhoneNumber` 开放能力获取用户手机号，再调用微信登录 API 完成登录。微信能力必须以后端真实接口和小程序配置为前提，不允许长期停留在前端 mock。

**验收标准**:
- WHEN 点击微信登录按钮 THEN 触发微信 `getPhoneNumber` 授权弹窗
- WHEN 用户授权成功 THEN 获取加密手机号数据，调用 `loginByWechat(code)` API
- WHEN 用户拒绝授权 THEN 提示"需要授权手机号才能登录"，不做其他操作
- WHEN 微信登录成功 THEN 与手机号登录相同的 Token 持久化流程
- WHEN 微信登录按钮 THEN 必须同样勾选用户协议才可使用
- WHEN 后端未配置微信小程序 `appId/appSecret` 或前端未显式启用该能力 THEN 页面显示“当前已对接手机号验证码登录 / 注册，微信登录稍后接入”提示，而不是继续走 mock 登录

### 3.6 用户协议勾选

**描述**: 页面底部显示用户协议勾选区域，包含复选框和"我已阅读并同意《用户协议》和《隐私政策》"文字。协议文字可点击跳转到对应 Web 页面。

**验收标准**:
- WHEN 页面初始加载 THEN 协议复选框默认未勾选
- WHEN 协议未勾选 THEN 登录按钮和微信登录按钮均禁用（视觉置灰）
- WHEN 用户勾选协议 THEN 登录按钮和微信登录按钮启用
- WHEN 用户点击"用户协议"文字 THEN 跳转到用户协议 H5 页面
- WHEN 用户点击"隐私政策"文字 THEN 跳转到隐私政策 H5 页面
- WHEN 未勾选协议时点击登录 THEN 提示"请先同意用户协议和隐私政策"并抖动复选框

### 3.7 登录后路由分发

**描述**: 登录成功后根据用户角色信息决定跳转目标。已选择身份的用户跳转到对应首页，未选择身份的新用户跳转到身份选择页。

**验收标准**:
- WHEN 登录成功且 `userInfo.role` 为 1（演员） THEN 跳转到首页 `/pages/home/index`（使用 `reLaunch`）
- WHEN 登录成功且 `userInfo.role` 为 2（剧组） THEN 跳转到首页 `/pages/home/index`（使用 `reLaunch`）
- WHEN 登录成功且 `userInfo.role` 为空/未设置 THEN 跳转到身份选择页 `/pages/role-select/index`（使用 `redirectTo`）
- WHEN 路由跳转 THEN 使用 `uni.reLaunch` 或 `uni.redirectTo` 确保登录页不留在页面栈中

## 4. 非功能需求

> 通用非功能需求见 [SHARED_CONVENTIONS.md](../SHARED_CONVENTIONS.md)

- **性能**: 验证码发送请求响应 < 2s。
- **安全**: 验证码 60s 倒计时防刷，登录按钮防重复点击，手机号不在前端日志中明文输出。

## 5. 约束条件

> 通用约束见 [SHARED_CONVENTIONS.md](../SHARED_CONVENTIONS.md)

- 组件库：使用 `00-02-shared-components` 中的 KpButton、KpInput 等组件
- API 层：使用 `00-03-shared-utils-api` 中的 `api/auth.ts` 和 `stores/user.ts`
- 微信登录依赖 `getPhoneNumber` 开放能力，需在微信后台配置
- 微信登录后端依赖小程序 `appId / appSecret` 与 `POST /api/auth/wechat-login`，未配置时只能判定为外部依赖阻塞，不能伪造成功能力
- 页面为非 Tab 页，使用自定义导航栏（`navigationStyle: custom`）
