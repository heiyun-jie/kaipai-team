# 00-35 后台工作台筛查范围可见化（Admin Dashboard Filter Scope Clarity）

> 状态：进行中 | 优先级：P1 | 依赖：00-34 admin-dashboard-overview-window-filter-alignment
> 记录目的：把工作台当前筛查影响范围做成显性信息，避免运营误读统计卡与最近事项的口径。

## 1. 背景

`00-34` 已为工作台接入 `dateFrom/dateTo/bizLine`，但当前页面仍只有筛查面板描述文字来说明口径差异：

- 时间窗口会影响统计卡与最近事项
- `bizLine` 只影响最近事项

这类边界如果只留在说明文案里，运营在切换筛查后仍可能把“最近事项被收窄”误认为“统计卡也被收窄”。工作台需要把当前筛查状态与影响范围显性展示出来。

## 2. 范围

### 2.1 本轮必须处理

- `kaipai-admin/src/views/dashboard/OverviewView.vue`

### 2.2 本轮不处理

- dashboard 后端接口
- 统计卡口径改造
- recent items 数据结构扩展

## 3. 需求

### 3.1 筛查范围提示

- **R1** 工作台必须显式展示“统计范围”和“最近事项范围”。
- **R2** 当时间窗口已设置时，范围提示必须展示当前时间窗口。
- **R3** 当 `bizLine` 已设置时，最近事项范围必须展示当前业务线。

### 3.2 口径边界

- **R4** 页面必须继续明确：`bizLine` 不影响统计卡。
- **R5** 页面不得让运营误读为“统计卡和最近事项始终共用完全相同的筛查口径”。

### 3.3 空态与跟进

- **R6** 最近事项空态应基于当前筛查条件给出更准确的提示，而不是固定文案。
- **R7** 范围提示与空态都应基于当前筛查状态实时更新。

## 4. 验收标准

- [ ] 已新增独立 Spec 并登记索引与代码映射
- [ ] 工作台已显式展示统计范围与最近事项范围
- [ ] `bizLine` 仅影响最近事项的边界在页面中保持清晰
- [ ] 最近事项空态可反映当前筛查条件
- [ ] `npm run build` 在 `kaipai-admin` 通过
