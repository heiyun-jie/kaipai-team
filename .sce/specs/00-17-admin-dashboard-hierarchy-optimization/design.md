# 00-17 设计说明

## 1. 设计原则

- 保持当前暖色后台风格，不切换整体品牌方向
- 用“块面、对比、边界”建立层级，而不是只靠字号变化
- 主按钮、状态标签、正文说明分别承担动作、判断、解释三种角色

## 2. 设计策略

### 2.1 顶部区域

- 顶部账号区改为更紧凑的账户卡，姓名与角色分层展示
- 服务状态提示保持轻量，不与主按钮争夺视觉中心

### 2.2 工作台概览

- 概览卡增加独立背景和强调边界
- 数字作为第一视觉锚点，指标名作为第二层，说明文字降级

### 2.3 模块卡

- 模块卡采用更清晰的头部、正文、底部操作区
- 状态标签放在头部，按钮进入底部独立操作带
- 对存在待处理事项的模块增加更强的强调样式

## 3. 影响文件

- `kaipai-admin/src/components/layout/AdminTopbar.vue`
- `kaipai-admin/src/components/business/StatusTag.vue`
