# 00-17 后台工作台层级优化（Admin Dashboard Hierarchy Optimization）

> 状态：已完成 | 优先级：P1 | 依赖：00-11 platform-admin-console
> 记录目的：解决后台工作台中状态、按钮和正文层级过于接近的问题，让运营能快速区分“看什么”和“做什么”。

## 1. 背景

当前后台工作台已经完成文案运营化，但视觉层级仍然偏平。概览数字、模块状态、进入按钮、账号区和页面说明处于近似权重，导致运营在扫描页面时难以快速定位重点。

## 2. 范围

### 2.1 本轮必须处理

- 工作台顶部区域与账号区的层级优化
- 概览指标卡的识别度优化
- 模块卡中状态、正文、按钮的分层
- 状态标签的可读性增强

### 2.2 本轮不处理

- 侧边导航结构重写
- 后台其他业务页面的大规模视觉重构
- 新增图表、拖拽看板或自定义仪表盘能力

## 3. 需求

### 3.1 信息层级

- **R1** 工作台必须把“概览指标”“模块状态”“执行动作”分成不同视觉层级
- **R2** 模块按钮需要显著区别于正文说明，不能与描述文字混在同一注意力层
- **R3** 状态标签需要一眼能识别当前模块是“待处理 / 运行中 / 可配置”

### 3.2 顶部区域

- **R4** 顶部账号区要清晰区分姓名、角色和头像，避免文字堆叠拥挤
- **R5** 顶部提示与页面主操作要保持明确主次关系，不抢工作台主内容注意力

### 3.3 工作台卡片

- **R6** 概览卡要突出数字与指标名称，辅助说明降级
- **R7** 模块卡要突出模块名称、当前状态和主操作按钮
- **R8** 卡片边界、背景和间距需要让每块信息自成区域，不再像铺平在大白板上

- [x] 工作台概览卡、模块卡和顶部账号区完成层级优化
- [x] 状态标签和主操作按钮可以被明显区分
