# 00-09 公共样式壳层抽离（Shared Style Shells）

> 状态：已完成（第一阶段） | 优先级：高 | 依赖：00-01, 00-08

## 1. 目标

在完成 Sass 入口治理后，继续对当前主线中的高重复样式壳层做低风险抽离，优先收敛“悬浮返回按钮”和“底部固定操作栏”两类跨页面重复样式，降低维护成本并控制样式残余继续扩散。

## 2. 本轮范围

- 返回按钮公共样式 mixin 抽离
- 底部固定操作栏公共样式 mixin 抽离
- 当前已命中的主线页面回接
- 构建验证与文档追溯

## 3. 需求

### 3.1 重复样式治理

- **R1** 样式治理必须优先处理跨页面高重复、低业务耦合的样式壳层，不得先做大面积视觉重写。
- **R2** 当前主线中重复的悬浮返回按钮样式必须抽离为公共 mixin，至少覆盖按钮外壳、箭头和文字三部分。
- **R3** 当前主线中重复的底部固定操作栏样式必须抽离为公共 mixin，至少覆盖操作栏容器、提示文案和主操作按钮壳层。
- **R4** 新增 mixin 必须允许页面保留少量个性化参数，不能为抽离而强迫所有页面视觉完全一致。

### 3.2 回接约束

- **R5** 回接时不得改变现有页面结构、路由和交互语义。
- **R6** 回接时不得引入新的全局样式污染，不得回退到 `@import`。
- **R7** 本轮优先覆盖当前主线与直接依赖页面，不强求一次性重构所有样式块。

### 3.3 验证

- **R8** 必须执行 `npm run type-check`
- **R9** 必须执行 `npm run build:mp-weixin`
- **R10** 必须执行 `npm run audit:steering`

## 4. 验收标准

- [x] 已将命中的重复返回按钮样式切换到公共 mixin
- [x] 已将命中的重复底部操作栏样式切换到公共 mixin
- [x] `npm run type-check` 通过
- [x] `npm run build:mp-weixin` 通过
- [x] `npm run audit:steering` 通过
