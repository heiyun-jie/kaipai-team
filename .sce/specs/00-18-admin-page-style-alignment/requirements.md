# 00-18 后台页面风格统一（Admin Page Style Alignment）

> 状态：已完成 | 优先级：P1 | 依赖：00-17 admin-dashboard-hierarchy-optimization
> 记录目的：把后台其他页面的布局、卡片、筛选区和详情区统一到工作台当前的风格语言，避免页面之间割裂。

## 1. 背景

工作台页面已经完成层级优化，但其他后台页面仍沿用较早的平面样式：页头较弱、筛选区像普通表单、表格主区与详情卡边界不明显，导致整体后台的使用体验不统一。

## 2. 范围

### 2.1 本轮必须处理

- 通用页头 `PageContainer`
- 通用筛选区 `FilterPanel`
- 表格主卡、详情卡、详情块等共用视觉壳层
- 让高频后台页面自动继承新的风格对齐

### 2.2 本轮不处理

- 业务字段、交互流程和接口契约调整
- 每个页面独立新增 hero 区块
- 侧边导航结构重构

## 3. 需求

### 3.1 统一布局

- **R1** 后台其他页面应沿用工作台当前的空间节奏、边界和卡片层级
- **R2** 页头需要清晰区分标题、说明和主操作区
- **R3** 筛选区需要像工具区而非正文表单

### 3.2 主内容区

- **R4** 表格主卡需要具备明确边界、留白和主要工作区感
- **R5** 详情卡和详情块要具备统一的卡片和信息块语言
- **R6** 分页区、详情操作区和多卡片拆分布局要有稳定的视觉规律

## 4. 验收标准

- [x] 已新增独立 Spec 并登记增量索引
- [x] `PageContainer`、`FilterPanel` 和共用卡片样式已统一到工作台风格
- [x] 多个后台业务页自动继承新的布局与 UI 风格
- [x] `npm run type-check` 与 `npm run build` 通过
