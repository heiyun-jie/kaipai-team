# 00-14 后台账号表单校验与高风险原因约束（Admin User Form Guard）

> 状态：进行中 | 优先级：P0 | 依赖：00-11 platform-admin-console, 00-13 admin-user-role-binding-guard
> 记录目的：补齐后台账号管理页在创建账号、编辑账号、绑定角色、重置密码、启停用等入口的前端表单校验与高风险原因约束，减少无效提交和审计缺口。

## 1. 背景

当前后台账号页已支持主要写操作，但仍存在两个缺口：

1. 创建 / 编辑 / 重置密码缺少明确的前端表单校验
2. 高风险操作的原因约束未完全统一，部分入口仍可空原因提交

## 2. 范围

### 2.1 本轮必须处理

- 创建后台账号表单校验
- 编辑后台账号表单校验
- 绑定角色原因约束
- 重置密码原因与交付方式约束
- 启用 / 禁用后台账号原因约束

### 2.2 本轮不处理

- 后端 DTO 校验规则变更
- 后台账号批量导入
- 更复杂的密码强度评分组件

## 3. 需求

### 3.1 基础表单校验

- **R1** 创建与编辑账号必须校验账号、姓名非空
- **R2** 创建账号必须校验初始密码非空
- **R3** 手机号与邮箱若填写，必须做基本格式校验
- **R4** 提交前必须统一去除字符串字段首尾空格

### 3.2 高风险操作原因约束

- **R5** 绑定角色必须填写原因
- **R6** 重置密码必须填写重置原因
- **R7** 重置密码必须填写凭证交付方式
- **R8** 启用 / 禁用后台账号必须填写原因

### 3.3 一致性

- **R9** 校验失败时必须在前端直接提示，不得把明显无效输入直接打到后端
- **R10** 所有约束不得改变后端现有请求结构

## 4. 验收标准

- [ ] 创建 / 编辑账号已补齐基础前端校验
- [ ] 绑定角色、重置密码、启停用已统一接入原因约束
- [ ] `npm run type-check` 与 `npm run build` 通过
