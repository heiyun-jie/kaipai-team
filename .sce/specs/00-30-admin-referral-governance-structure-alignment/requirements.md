# 00-30 后台邀请治理结构优化（Admin Referral Governance Structure Alignment）

> 状态：进行中 | 优先级：P1 | 依赖：00-18 admin-page-style-alignment，00-20 admin-filter-inline-alignment，05-12 share-invite-code-consolidation
> 记录目的：把 referral 模块从“表格直出”推进到“治理视图”，补齐治理摘要和时间窗口筛查能力。

## 1. 背景

`05-12` 已经完成 referral 域第一阶段口径收口，明确后台邀请模块属于治理域，而不是前台分享入口。

但当前后台页面仍存在两个结构问题：

- `邀请记录`、`异常邀请` 缺少治理摘要，运营打开页面后只能直接读表格，无法先看到当前查询窗口下的治理态势。
- `邀请记录`、`异常邀请`、`邀请资格` 虽然查询类型已预留时间字段，但页面没有完整暴露时间窗口筛查，复盘和排查成本偏高。

## 2. 范围

### 2.1 本轮必须处理

- `kaipai-admin/src/views/referral/RecordsView.vue`
- `kaipai-admin/src/views/referral/RiskView.vue`
- `kaipai-admin/src/views/referral/EligibilityView.vue`
- 共享治理摘要卡壳层

### 2.2 本轮不处理

- 后端 referral API
- referral 数据模型、字段命名、状态流转
- `PoliciesView.vue` 的规则表单结构重做
- 新增独立统计接口

## 3. 需求

### 3.1 治理摘要

- **R1** `邀请记录` 页面必须在筛选区前展示治理摘要卡，至少包含查询规模、当前页有效记录、当前页风险命中。
- **R2** `异常邀请` 页面必须在筛选区前展示治理摘要卡，至少包含查询规模、当前页待处置记录、当前页复核完成或已解除样本。
- **R3** 治理摘要卡应使用共享展示壳层，不允许两页继续复制样式。

### 3.2 时间窗口筛查

- **R4** `邀请记录` 页面必须补齐 `注册时间`、`生效时间` 时间窗口筛查。
- **R5** `异常邀请` 页面必须补齐 `注册时间` 时间窗口筛查。
- **R6** `邀请资格` 页面必须补齐 `生效时间` 时间窗口筛查。

### 3.3 交互边界

- **R7** 本轮新增筛选仅装配已有 query 字段，不新增接口参数。
- **R8** 重置动作必须同步清空新增时间窗口。
- **R9** 页面说明文案应继续维持“治理后台”语义，不回退到“裂变入口”语义。

## 4. 验收标准

- [ ] 已新增独立 Spec 并登记索引与代码映射
- [ ] `邀请记录`、`异常邀请` 已出现治理摘要卡
- [ ] `邀请记录`、`异常邀请`、`邀请资格` 已可按时间窗口筛查
- [ ] `npm run build` 在 `kaipai-admin` 通过
