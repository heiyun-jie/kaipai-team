# 00-20 后台筛选区横向对齐（Admin Filter Inline Alignment）

> 状态：已完成 | 优先级：P1 | 依赖：00-18 admin-page-style-alignment, 00-19 admin-verify-page-refinement
> 记录目的：统一后台筛选区的标签与输入控件布局，确保提示文本与输入框保持同一行且垂直居中。

## 1. 背景

实名认证审核页二次优化后，筛选项被调整为标签上置结构。但当前后台其他筛选页仍保持“标签 + 输入同一行”的工作台习惯，导致实名认证页与其他运营页不一致，也不符合运营同学对筛选效率的预期。

## 2. 范围

### 2.1 本轮必须处理

- 共享 `FilterPanel` 中所有行内筛选表单的标签与输入对齐规则
- 实名认证审核页筛选区回收为横向对齐结构
- 保证后台页筛选项在桌面端保持同一行、垂直居中

### 2.2 本轮不处理

- 非筛选区的表格、详情抽屉和概览卡重做
- 后端接口、查询字段与业务流程
- 小屏极窄场景的独立移动端重构

## 3. 需求

### 3.1 对齐规则

- **R1** 筛选标签与输入控件必须同一行展示，不允许默认上下堆叠
- **R2** 标签文本、输入内容、下拉箭头和清除图标需要垂直居中
- **R3** 行内筛选项在多列排列时应保持稳定间距，不出现高低错位

### 3.2 页面一致性

- **R4** 实名认证审核页应回到与异常邀请等后台页一致的筛选结构
- **R5** 共享筛选壳层应承担这套对齐规则，避免逐页散落覆盖

- [x] 实名认证审核页筛选区已恢复横向对齐并保持居中
