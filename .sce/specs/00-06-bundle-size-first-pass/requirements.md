# 00-06 首批减体积优化（Bundle Size First Pass）

> 状态：进行中 | 优先级：高 | 依赖：00-05, 05-07

## 1. 目标

在不改变当前主线业务行为的前提下，执行一轮低风险、可验证、可持续的前端减体积优化，优先清理会额外生成公共聚合产物的代码组织方式。

## 2. 本轮范围

- 组件 barrel 聚合引用治理
- 共享组件直接引用收敛
- 构建产物与包体审计对比验证

## 3. 需求

### 3.1 组件引用方式

- **R1** 页面和模块中不再通过 `@/components` barrel 聚合入口引用共享组件。
- **R2** 共享组件应改为直接引用具体文件，避免生成额外的组件聚合产物。
- **R3** 若移除 barrel 入口后仓库无剩余依赖，必须同步删除失效聚合文件，不留残余。

### 3.2 行为边界

- **R4** 本轮优化不得改变任何页面入口、路由、会员判断、分享路径和交互语义。
- **R5** 本轮优化不得引入新的业务功能或视觉改动。

### 3.3 验证

- **R6** 必须执行 `npm run type-check`
- **R7** 必须执行 `npm run build:mp-weixin`
- **R8** 必须执行 `npm run audit:mp-package`
- **R9** 必须对比本轮优化前后的产物结构变化，至少确认 `components/index.js` 是否已消失或不再生成。

## 4. 验收标准

- [ ] `@/components` 聚合引用已从当前主线移除
- [ ] 失效聚合文件已清理
- [ ] 类型检查通过
- [ ] 微信小程序构建通过
- [ ] 包体审计通过
