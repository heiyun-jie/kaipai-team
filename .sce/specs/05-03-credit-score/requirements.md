# 05-03 信用积分与演员等级（Credit & Level）

> 状态更新：当前产品阶段搁置。当前分支以 `05-05-card-share-membership` 为主线，本 Spec 仅作历史保留。

> 状态：预留 | 优先级：中 | 依赖：00-03, 05-02

## 1. 功能概述

信用积分制：从 **0 分起步**，通过正向行为累积积分（满分 100），通过违规行为和不活跃衰减扣除积分。

积分直接决定演员等级和信用徽章。一个数据源，两种展示形态。

**系统边界**：积分系统只负责「收到信号 → 计算分数 → 输出等级」。爽约/迟到判定、差评标准、投诉审核由平台运营层提供输入。

## 2. 积分模型（0→100）

### 2.1 积分来源

#### A. 档案建设（满分 30）

| 行为 | 积分 | 说明 |
|------|------|------|
| 上传头像 | +3 | 一次性 |
| 完善基本信息（姓名+性别+年龄+身高+城市全填） | +5 | 一次性 |
| 上传照片（≥3 张且覆盖 ≥2 个分类） | +5 | 一次性 |
| 上传视频简历 | +5 | 一次性 |
| 填写自我介绍（≥50 字） | +4 | 一次性 |
| 添加作品经历（≥1 条） | +5 | 一次性 |
| 完善形象标签（体型+发型+语言至少各选 1） | +3 | 一次性 |

#### B. 拍摄经验（满分 30）

| 里程碑 | 积分 | 说明 |
|--------|------|------|
| 首次完成拍摄 | +10 | 里程碑 |
| 累计完成 5 次 | +5 | 里程碑 |
| 累计完成 15 次 | +5 | 里程碑 |
| 累计完成 30 次 | +5 | 里程碑 |
| 参与 ≥3 种不同类型项目 | +5 | 里程碑 |

#### C. 口碑评价（满分 25）

| 成就 | 积分 | 说明 |
|------|------|------|
| 首次获得好评 | +5 | 里程碑 |
| 好评率 ≥ 80% | +5 | 动态（降到 80% 以下则扣回） |
| 好评率 ≥ 90% | +5 | 动态（降到 90% 以下则扣回） |
| 连续 5 次无爽约 | +5 | 里程碑 |
| 连续 10 次无爽约 | +5 | 里程碑 |

#### D. 活跃贡献（满分 15）

| 行为 | 积分 | 说明 |
|------|------|------|
| 首次投递 | +3 | 里程碑 |
| 累计投递 10 次 | +3 | 里程碑 |
| 近 30 天有登录 | +3 | 动态（30 天无登录则扣回） |
| 近 30 天有投递 | +3 | 动态（30 天无投递则扣回） |
| 分享过名片 | +3 | 一次性 |

**合计满分**：30 + 30 + 25 + 15 = **100 分**

### 2.2 积分扣减体系（三层）

积分扣减分三层，最低扣至 0 分。

#### Layer 1：动态回收（系统自动，条件恢复后自动回来）

| 场景 | 扣回 | 恢复条件 |
|------|------|---------|
| 好评率掉到 90% 以下 | -5 | 好评率恢复 ≥90% |
| 好评率掉到 80% 以下 | -5 | 好评率恢复 ≥80% |
| 30 天无登录 | -3 | 重新登录 |
| 30 天无投递 | -3 | 重新投递 |
| 档案完整度降低（删照片到 <3 张、删视频、清空介绍等） | 扣回对应一次性积分 | 重新补全 |

#### Layer 2：惩罚性扣除（三级）

**轻微 — 系统自动判定：**

| 行为 | 扣分 |
|------|------|
| 迟到（确认到场但晚到） | -5 |
| 30 天内取消已确认投递 ≥3 次 | -3 |

**中重度 — 投诉处理流程（剧组/平台发起投诉 → 平台审核确认 → 执行扣分）：**

| 行为 | 扣分 |
|------|------|
| 爽约未到场 | -10 |
| 获得差评 | -5（连续差评额外 -3） |
| 未按要求完成拍摄任务 | -8 |
| 拍摄态度消极（剧组反馈） | -5 |
| 拍摄中途退出（无正当理由） | -15 |
| 投诉成立（骚扰、不专业行为） | -15 |
| 资料造假被确认 | -20 |

**极端 — 平台直接处理：**

| 行为 | 后果 |
|------|------|
| 累计中重度投诉 ≥3 次 | 信用冻结，降至 LV.1，限制投递 |
| 恶意行为（暴力、严重骚扰） | 直接封号，不走积分体系 |

**累犯递增规则**：
- 同类违规 30 天内第 2 次：扣分 ×1.5
- 同类违规 30 天内第 3 次：扣分 ×2 + 投递冷却期 7 天

#### Layer 3：时间衰减（防躺平）

| 条件 | 衰减速率 | 衰减池 |
|------|---------|--------|
| 连续 60 天无拍摄完成 | 每 30 天 -2 | 仅从「拍摄经验」「活跃贡献」池扣 |

- 恢复活跃后衰减立即停止
- 不触及档案建设分和里程碑分
- 衰减保底线 = 档案建设得分

### 2.3 积分性质说明

- **一次性积分**：达成后永久保留（档案完整度降低时动态回收，补全后恢复）
- **里程碑积分**：达成后永久保留，不受衰减影响
- **动态积分**：条件不满足时自动扣回，条件恢复后自动恢复
- **惩罚扣分**：从总积分中扣除，只能通过正向行为重新赚回
- 任何扣减后积分不低于 0

### 2.4 保护机制

**新人保护期**：
- 注册后首 30 天为保护期
- 惩罚性扣分减半
- 不触发累犯递增
- 严重违规（-15 以上）不受保护

**降级缓冲**：
- 等级下降时给 7 天缓冲期
- 缓冲期内恢复到阈值以上 → 取消降级
- 严重违规导致的降级无缓冲

**投诉申诉**：
- 中重度扣分可在 7 天内申诉
- 申诉期间扣分生效但标记"申诉中"
- 申诉成功 → 积分恢复 + 记录标注"已撤销"
- 每 30 天最多申诉 2 次

### 2.5 低分后果阶梯

| 积分 | 后果 |
|------|------|
| < 10 | 投递时显示"信用待提升"标签（剧组可见） |
| < 5 | 每日投递上限降为 3 个 |
| = 0 | 冻结投递 7 天，只能做信用修复任务 |

### 2.6 信用修复机制

积分跌到 20 以下时，系统推送"信用修复指引"：

| 修复方式 | 恢复 | 限制 |
|---------|------|------|
| 补全因删除而失去的档案项 | 恢复对应一次性积分 | 仅限之前拥有后失去的 |
| 连续 7 天登录 | +1 修复分 | 每季度最多 3 分，总修复分不超过 30 |

## 3. 演员等级（积分区间映射）

| 积分区间 | 等级 | 名称 | 标签 | 徽章 |
|---------|------|------|------|------|
| 0-9 | LV.1 | 新秀演员 | NEWCOMER | — |
| 10-24 | LV.2 | 潜力演员 | POTENTIAL | — |
| 25-44 | LV.3 | 活跃演员 | ACTIVE | 🥉 |
| 45-59 | LV.4 | 认证演员 | CERTIFIED | 🥈 |

等级随积分实时变化，升降即时生效。

| 等级 | 权益 |
|------|------|
| LV.1-2 | 基础功能，无特殊权益 |

| LV.5 | AI 简历润色每日次数从 10 次提升至 20 次 |
| LV.6 | 高质量通告优先推送通知 |
| LV.7 | 平台"王牌演员"专属推荐位 |

### 3.2 排名权益

| 排名 | 权益 |
|------|------|
| TOP 3 | 排行榜页面领奖台展示 + 首页"信用之星"展示位 |
| TOP 10 | 首页"信用之星"展示位（头像+姓名+等级） |

## 5. 需求清单

- **R1** 积分从 0 开始，根据行为规则叠加

## 7. 数据结构

```typescript
// 积分总览
interface CreditScore {
  totalScore: number          // 当前总积分 0-100
  level: number               // 等级 1-7
  levelTitle: string          // "新秀演员"..."王牌演员"
  levelTag: string            // "NEWCOMER"..."ACE ACTOR"
  breakdown: CreditBreakdown  // 四类明细
  nextLevelHint?: string      // 升级提示
  nextLevelScore?: number     // 下一等级最低分
  status: 'normal' | 'frozen' // 正常/冻结
  isNewbie: boolean           // 是否在新人保护期内
  decayActive: boolean        // 是否正在衰减
}

// 四类明细
interface CreditBreakdown {
  profile: CategoryScore      // 档案建设 0-30
  experience: CategoryScore   // 拍摄经验 0-30
  reputation: CategoryScore   // 口碑评价 0-25
  activity: CategoryScore     // 活跃贡献 0-15
}

interface CategoryScore {
  earned: number              // 已得分
  max: number                 // 满分
  items: ScoreItem[]          // 具体得分项
}

interface ScoreItem {
  label: string               // "上传头像"
  score: number               // 该项分值
  achieved: boolean           // 是否已达成
  type: 'once' | 'milestone' | 'dynamic'
}

// 积分记录
interface CreditRecord {
  id: number
  type: 'earn' | 'deduct' | 'revoke' | 'complaint' | 'decay'
  // 获得 / 轻微自动扣除 / 动态扣回 / 投诉扣除 / 时间衰减
  amount: number
  reason: string
  category: 'profile' | 'experience' | 'reputation' | 'activity'
  totalAfter: number          // 变化后总分
  createdAt: string
  // 投诉相关（type='complaint' 时有值）
  complaintId?: number
  appealStatus?: 'none' | 'pending' | 'approved' | 'rejected'
  // none=未申诉 pending=申诉中 approved=申诉成功(已撤销) rejected=申诉驳回
}

// 投诉记录（投诉处理流程产出）
interface CreditComplaint {
  id: number
  actorId: number
  source: 'crew' | 'platform'         // 剧组投诉 / 平台发起
  severity: 'moderate' | 'severe'      // 中度 / 严重
  reason: string                       // 投诉原因描述
  deductAmount: number                 // 扣分值
  status: 'pending' | 'confirmed' | 'dismissed'  // 待审核/确认/驳回
  appealStatus: 'none' | 'pending' | 'approved' | 'rejected'
  projectId?: number
  roleId?: number
  createdAt: string
  confirmedAt?: string
}

// 排行榜
interface CreditRankItem {
  rank: number
  userId: number
  name: string
  avatar: string
  totalScore: number
  level: number
  levelTitle: string
  isCurrentUser: boolean
}
```

## 10. 验收标准

**加分：**
- [ ] 积分从 0 开始，档案建设行为正确加分
- [ ] mock 阶段积分和等级与档案完整度对应（LV.1-3）
- [ ] 一次性/里程碑积分达成后永久保留

**扣减 — 动态回收：**
- [ ] 好评率、登录、投递条件不满足时自动扣回，恢复后自动回来
- [ ] 档案完整度降低时回收对应积分，补全后恢复

**扣减 — 惩罚性扣除：**
- [ ] 轻微违规（迟到、频繁取消）系统自动扣分
- [ ] 中重度违规走投诉处理流程：投诉 → 审核 → 确认后才执行扣分
- [ ] 累犯递增：30 天内同类第 2 次 ×1.5，第 3 次 ×2 + 冷却 7 天
- [ ] 扣分后积分不低于 0

**扣减 — 时间衰减：**
- [ ] 连续 60 天无拍摄后每 30 天 -2，仅从拍摄经验和活跃池扣
- [ ] 衰减不低于档案建设得分，恢复活跃后立即停止

**保护机制：**
- [ ] 新人保护期 30 天内惩罚性扣分减半（严重违规除外）
- [ ] 降级缓冲 7 天，缓冲期内恢复积分可取消降级
- [ ] 中重度扣分可在 7 天内申诉，申诉成功后积分恢复

**低分后果：**
- [ ] 积分 <10 显示"信用待提升"标签
- [ ] 积分 <5 每日投递上限 3 个
- [ ] 积分 =0 冻结投递 7 天
